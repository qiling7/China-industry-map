# 移动端响应式适配 — 设计方案

> 日期：2026-05-08 | 状态：待实施

## 一、断点策略

| 屏幕宽度 | 设备类型 | 布局方案 |
|----------|----------|----------|
| `< 768px` | 手机 | 全新竖屏布局（全屏地图 + 浮动搜索 + 左侧抽屉） |
| `>= 768px` | 平板 / 桌面 | 保持现有 Grid 布局（地图 + 右侧固定面板） |

## 二、手机端布局（< 768px）

### 默认状态：全屏地图

- 地图占满 100vw × 100vh
- 搜索栏浮动在地图顶部，半透明白色背景
- 左侧边缘"索引"把手，竖排文字，最小 44×44px 触控区域
- 地图标记与联动功能正常运作

### 面板展开：左侧抽屉

- **触发方式**：点击把手 或 从屏幕左边缘右滑
- **面板宽度**：约 75vw，地图同步右移保留视野
- **关闭方式**：点击右侧地图区域 或 面板左滑收起
- **面板内容**：搜索框 + 分类下拉 + 重置按钮 + 结果卡片列表
- **过渡动画**：CSS transform/transition，面板和地图同步滑动

## 三、搜索与过滤（手机版）

搜索控件位于抽屉面板内部顶部：
- 输入框 + 分类下拉 + 重置按钮，布局与桌面版一致
- 输入实时过滤结果列表
- 过滤时非匹配标记半透明，匹配标记保持不透明
- 重置恢复全量数据

## 四、双向联动（保留完整功能）

- **卡片 → 地图**：点击卡片 → 面板收起 + `flyTo` 飞行 + `highlightMarker` 呼吸灯
- **地图 → 卡片**：点击标记 → 面板展开 + 结果滚动定位 + 卡片高亮

## 五、触屏优化要求

- 卡片间距加大，点击区域 ≥ 44px 高
- 面板把手触控区域 ≥ 44×44px
- 防止地图双击缩放与面板手势冲突
- 搜索输入框字体 ≥ 16px（防止 iOS 自动缩放）

## 六、修改范围

### 需修改的文件
- `index.html` — 增加抽屉 DOM 结构（`#drawer` 面板 + `#drawer-handle` 把手）
- `css/style.css` — 新增 `@media (max-width: 767px)` 媒体查询块

### 需新增的文件
- `js/ui/drawer.js` — 抽屉开关逻辑、手势监听、展开/收起动画

### 需微调的文件
- `js/ui/panel.js` — 适配移动端面板交互（展开/收起时触发对应逻辑）
- `js/main.js` — 移动端初始化时绑定抽屉而非桌面面板

### 无需改动的文件
- `js/config.js`、`js/map/map.js`、`js/map/markers.js`、`js/filter/search.js`、`js/filter/render.js`、`js/ui/card.js`、后端所有文件

## 七、技术实现要点

- **纯 CSS 响应式**：通过 `@media` 查询切换布局，不引入额外框架
- **ES6 模块**：`drawer.js` 作为独立模块，与现有模块通过明确接口通信
- **CSS 过渡**：面板使用 `transform: translateX()` 实现滑入滑出
- **无第三方依赖**：手势通过原生 `touchstart/touchend` 或 `pointerdown/pointerup` 实现
